# 操作系统的进程调度实验

## PCB数据结构

PCB数据结构尽量全面，方便满足不同调度算法的设计需求

- （pid）进程标识数
- （plink）就绪队列链指针（初始化为null）
- （ppri）优先数（越高优先级越高）/轮转时间片数（伪随机初始化）
- （cur_cost）当前占用CPU时间片数
- （pcost）进程所需时间片数（伪随机初始化）
- （pstat）进程状态（R运行，W就绪，F完成）

## 调度算法

### 优先数

pri_schedule()

- 维护一个vector<PCB>

- 首先按照优先数大小降序排列
- 每次运行一个时间片，减少优先数，排序，循环。

### 简单轮转

simple_schedule()

- 按先后次序排列
- 顺序执行
